class 7:Backend Development

Server, Hosting & Deployment

Server: computer or software application that provides services or resources to other computers, known as clients, within a network

Hosting: service of providing the infrastructure, resources, and environment for a website, application, or service to be accessible on the internet.

Deployment:process of making a software application or system available for use



Server-side versus client-side code

Client-side: code that is run on the user's computer — when a web page is viewed, the page's client-side code is downloaded, then run and displayed by the browser.

Server-side: code run on the server, then its results are downloaded and displayed in the browser

class 10:Data storage and databases

Topics

Datastorage: Data storage refers to the physical or digital means of preserving and retaining data for future use. It is the broader concept that encompasses various methods and technologies for storing data.
Data storage can include a wide range of storage technologies, such as hard disk drives (HDDs), solid-state drives (SSDs), cloud storage, tape storage, optical storage, and more.

Databases: a structured collection of data organized for efficient storage, retrieval, and management. Designed to support the efficient querying and processing of data, enforcing relationships between different pieces of information.

Server, Database, Hosting and Deployment



types of database

Rational Databases (RDBMS) 

- Organize data into tables with rows and columns
- Enforce relationships between tables
- Examples: MySQL, PostgreSQL, Oracle Database, Microsoft SQL Server

No SQL Databases 

- Designed for flexibility and scalabilit.
- Do not rely on a fixed schema

In-Memory Databases 

- Store data in the system's main memory (RAM) for faster access

Distributed Databases 

- Distribute data across multiple nodes or servers for improved scalability and fault tolerance


Excercise/Research: SQL

Database Language: SQL is a language used for managing and manipulating relational databases.

Queries and Commands: It allows users to write queries and commands to interact with databases, performing tasks like retrieving, inserting, updating, and deleting data.

Structured Format: SQL follows a structured format with common keywords such as SELECT, INSERT, UPDATE, and DELETE, making it easy to communicate with and manage databases.
